# SekaiToolsMauiText 平台模式设计说明

> **DEPRECATED：MAUI 路线已废弃。现行方案见 [`avalonia-merge-plan.md`](./avalonia-merge-plan.md)。**

## 目标

`SekaiToolsMauiText` 需要从“旧版本地翻译工具”演进成“新一代跨平台翻译工作台”。

该工作台同时支持：

- 本地文件模式
- `SekaiPlatform` 平台模式

## 设计原则

### 1. 一个页面，两种来源

不维护两套独立编辑器页面。

统一策略：

- 平台原文行和本地剧本行都映射成统一的行模型
- 对话行继续使用 `TranslateLineDialog`
- 非对话行继续使用 `TranslateLineEffect`

### 2. 平台元信息只加在行模型上

平台模式需要的字段：

- `source_line_id`
- `line_no`
- `line_type`

这些字段不应该污染旧的本地翻译文本格式，只应保存在运行时模型上。

### 3. 上传走平台 DTO，不走本地 txt 序列化

本地保存：

- 继续使用旧的 `txt` 输出格式

平台上传：

- 把当前行模型转换成 `CreateTranslationVersionRequest`
- 不依赖导出的 `txt`

## 当前页面结构

页面分为四块：

1. `SekaiPlatform` 登录区
2. 平台剧情浏览区
3. 本地兼容模式区
4. 通用翻译编辑区

## 当前实现状态

### 已有

- 平台地址配置
- 登录 / 登出 / 刷新会话
- 租户切换
- 剧情类型 / 剧情集 / 剧情 / 版本 UI
- 平台原文行映射为编辑行
- 平台译文版本加载
- 平台对照版本加载
- 平台新版本上传请求构造
- 本地剧本 / 翻译 / 对照 / 导出保留
- 页面流程开始抽离到服务层

### 未完成

- 平台模式下更完整的空状态和禁用状态控制
- 更细的故事导航体验
- 自动恢复
- 文本检查辅助

## 本地模式与平台模式的状态边界

### 本地模式进入条件

- 用户载入本地剧本文件

### 平台模式进入条件

- 用户从平台加载剧情原文

### 切换规则

- 进入平台模式时，应清空本地剧本路径和本地翻译路径
- 进入本地模式时，应清空当前平台剧情上下文
- 两种模式不共享“保存目标”

## 上传策略

一期只支持：

- 创建新版本

一期不支持：

- 覆盖已有版本
- 编辑远端版本后回写同一版本

原因：

- 与平台版本归档模型一致
- 风险更低
- 更容易做审计

## 后续演进建议

### 1. 抽离页面服务

当前仍有一部分流程在 `TranslatePage.xaml.cs` 中，但平台会话、平台剧情浏览和本地工作区逻辑已经开始抽离。

后续建议抽出：

- Translation Upload Service
- Platform View State Coordinator

### 2. 抽离模式状态

后续可把“本地模式 / 平台模式”显式抽成状态对象，减少页面代码中的条件分支。

### 3. 迁移新 SekaiText 的编辑体验

优先迁：

- 更细的导航
- 更稳定的版本载入逻辑
- 更明确的状态提示

再迁：

- 文本校验
- 恢复逻辑
- 比较能力
